# Experiência do Usuário

## Filosofia

O Nyx OS deve parecer menos uma ferramenta corporativa e mais uma central pessoal viva.

A experiência deve ser clara, direta e com identidade, mas sem sacrificar usabilidade.

## Sensação principal

O usuário deve sentir:

> “Eu posso parar de segurar tudo na cabeça. O sistema segura comigo.”

## Modelo de interação

O Nyx OS deve permitir três formas principais de uso:

1. **Visual** — dashboard, listas, cards, projetos e contexto.
2. **Rápida** — captura de dados com pouco atrito.
3. **Assistida** — Nyx consultando, resumindo e operando dados.

## Dashboard

O dashboard não deve ser apenas bonito.

Ele deve responder:

- O que exige minha atenção hoje?
- O que ficou pendente?
- Quais projetos estão ativos?
- Qual próximo passo faz sentido?
- O que a Nyx sabe que pode me ajudar agora?

## Personalidade e presença

A personalidade da Nyx pode aparecer na experiência, mas o produto não deve depender de efeitos visuais ou teatralidade para funcionar.

Primeiro vem utilidade. Depois vem charme.

## Anti-atrito

O sistema deve aceitar que o usuário pode esquecer, pausar, mudar de foco, chegar cansado ou retomar depois de dias.

Por isso, a experiência deve favorecer:

- retomada de contexto;
- históricos claros;
- próximos passos simples;
- poucos cliques para capturar;
- documentação de decisões.
