# Visão de Longo Prazo

## Direção geral

O Nyx OS deve evoluir de uma aplicação web para uma infraestrutura pessoal modular.

A visão de longo prazo inclui rodar o projeto em um micro PC ou servidor dedicado, funcionando como base local para dados, automações, integrações e assistentes.

## Futuras capacidades possíveis

- Nyx Local rodando em ambiente próprio.
- Integração com Obsidian.
- Integração com Home Assistant.
- Dashboard para quarto/casa.
- Controle por voz.
- Avatar ou representação visual.
- Automação de rotinas.
- Integração com Workflow Lens.
- Relatórios automáticos.
- Memória local persistente.
- Módulos especializados por área da vida.

## Relação com micro PC / servidor

A ideia futura é que o Nyx OS possa deixar de ser apenas uma aplicação hospedada e passe a ser uma central local.

Isso permitiria:

- maior controle sobre dados;
- integrações locais;
- automações domésticas;
- funcionamento independente de serviços externos;
- conexão mais profunda com Nyx Local.

## Como essa visão afeta o presente

Mesmo que o MVP seja simples, decisões atuais devem evitar caminhos sem saída.

Por isso:

- preferir arquitetura modular;
- separar dados, UI e APIs;
- documentar contratos;
- evitar acoplamento excessivo com um único provedor;
- manter clareza entre produto atual e sonho futuro.

## Regra importante

A visão de longo prazo inspira o projeto, mas não deve engolir o MVP.

O MVP prova valor. A visão guia crescimento.


## Migração futura para ambiente próprio

A visão de longo prazo é que, quando a Nyx Assistente for movida para um mini PC ou mini servidor, o Nyx OS vá junto.

Nesse cenário, o Nyx OS deixa de depender principalmente de serviços pagos em nuvem e passa a funcionar como uma infraestrutura pessoal controlada pelo usuário.

Essa decisão não muda o MVP. Por agora, o foco é construir a base do produto sem gastar energia com infraestrutura local prematura.
