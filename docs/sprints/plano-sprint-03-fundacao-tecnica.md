# Sprint 03 — Fundação Técnica e Setup do Projeto

## Objetivo

Criar a fundação técnica definitiva do Nyx OS sem implementar funcionalidades de produto.

## Escopo

- Definir stack oficial.
- Organizar o monorepo.
- Definir packages planejados.
- Documentar arquitetura de dependências.
- Documentar padrões de código, Git, configuração, logs, erros e testes.
- Preparar CI inicial.
- Registrar ADRs técnicas.
- Aplicar o workflow obrigatório de Pull Request ao final da sprint.

## Fora de escopo

- Dashboard novo.
- Captura rápida.
- Nyx Local implementada.
- APIs finais.
- Automação real.
- Qualquer funcionalidade de produto.

## Entregáveis

- `docs/arquitetura/fundacao-tecnica.md`
- `docs/workflow/desenvolvimento.md`
- `docs/workflow/qualidade.md`
- `docs/workflow/configuracao.md`
- ADRs técnicas da Sprint 03.
- Estrutura `apps/`, `packages/`, `configs/`, `infrastructure/`, `docker/`, `assets/` e `tests/`.
- Pull Request aberta seguindo o padrão oficial do projeto.

## Critério de sucesso

A sprint é bem-sucedida se um novo desenvolvedor ou agente conseguir entender:

- onde cada tipo de código deve viver;
- quais tecnologias são oficiais;
- como dependências entre módulos devem funcionar;
- como configurar, testar e abrir PR;
- quais decisões técnicas já foram tomadas.
