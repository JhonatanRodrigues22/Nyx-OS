# Sprint 01 — Fundação Operacional do Nyx OS

## Objetivo

Construir a primeira base funcional do Nyx OS sem tentar implementar toda a visão de longo prazo.

A Sprint 01 deve provar que o sistema consegue nascer como uma aplicação real, com estrutura, domínio inicial e caminho claro para captura e dashboard.

## Escopo recomendado

- Estrutura base da aplicação.
- Configuração inicial de autenticação e banco.
- Tipos centrais do domínio.
- Entidades iniciais: tarefa, projeto, nota, decisão, memória e check-in.
- Primeiras rotas ou serviços de dados.
- Layout inicial do dashboard/cockpit.
- Captura rápida em versão mínima, se possível.

## Ordem oficial

A captura rápida é parte essencial do MVP, mas pode ser implementada em camadas.

Para resolver a ambiguidade anterior:

1. Sprint 01 cria a fundação e pode incluir uma captura mínima.
2. Sprint 02 amadurece dashboard/cockpit e visualização do dia.
3. Sprint 03 aprofunda captura, organização e relações entre entidades.

Assim, captura não é adiada como conceito, mas também não precisa nascer completa na primeira sprint.

## Fora de escopo

- Nyx Local.
- Deploy em mini PC.
- Voz.
- Avatar.
- Home Assistant.
- Automações complexas.
- IA autônoma executando ações.
- Analytics avançado.

## Critério de sucesso

A Sprint 01 é bem-sucedida quando:

- o projeto roda;
- existe uma base de domínio clara;
- dados iniciais podem ser criados ou preparados;
- o dashboard tem esqueleto funcional;
- a documentação continua consistente com o código.
